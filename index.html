FAQ

Files & data

Which file types work?

Copilot needs an .xlsx or .xlsm file. It won't run on .xls or .csv — open the file and use Save As to make it .xlsx first.

Where does the file need to live?

Saved in OneDrive or SharePoint with AutoSave on. On a file stored only on your PC, the Copilot button is greyed out.

How should the data be laid out?

As one clean Table — select it and press Ctrl+T, with a header row. No merged cells, no blank rows, nothing hidden.

Limits & security

How much data can it handle?

Around 2 million cells per analysis. Keep to one clean table; for more, filter or split it with Power Query.

Can it read several tabs at once?

Standard chat focuses on one table. Agent Mode can work across multiple sheets in the workbook.

Why did it stop responding?

There are fair-use limits (a few hundred requests an hour). Pause for a moment, then try again.

Is our data safe?

It stays inside your Microsoft 365 tenant's compliance boundary — your prompts and data aren't used to train the AI models.

Choosing a tool

Agent Mode vs the Analyst agent — which does what?

Agent Mode works inside Excel with native building blocks — combining, cleaning, formulas, PivotTables and charts — and edits your sheet. The Analyst agent (in the Microsoft 365 Copilot app) reasons over the data and runs Python for modelling — forecasts, trends and segmentation — showing its working. Rule of thumb: build or reshape the sheet → Agent Mode; forecast, trend or segment → the Analyst agent. (Excel does have a native Forecast Sheet for a quick trend, but for a real, explained forecast use the Analyst agent.)

Figures like file size and request limits can change as Microsoft updates Copilot in Excel.
